# i18n Design — Clipboard Manager

**Date:** 2026-04-23  
**Status:** Approved

---

## Overview

Add runtime language switching (English / German) to the clipboard manager frontend. The selected language is persisted to the SQLite database like other settings. No separate build per language — both translations ship in one bundle.

---

## Architecture

### Library

Use `@ngx-translate/core` for runtime translation. **Do not** use `@ngx-translate/http-loader` or Angular's built-in `@angular/localize` (the latter is compile-time only and incompatible with runtime switching).

### Translation Files

Translations live in typed TypeScript files, not JSON:

```
src/app/i18n/
  translation.interface.ts   ← shared Translation interface (schema enforcement)
  en.ts                      ← English translations (implements Translation)
  de.ts                      ← German translations (implements Translation)
  translate-loader.ts        ← custom TranslateLoader that imports the TS files directly
```

TypeScript enforces that both `en.ts` and `de.ts` satisfy the `Translation` interface — missing or mistyped keys are compile-time errors. No HTTP fetch required.

### I18nService

A thin `I18nService` wraps `@ngx-translate`'s `TranslateService` and owns all language-switching logic:

- On init: reads persisted language from `SettingsService`; if absent (`null`), falls back to `navigator.language.startsWith('de') ? 'de' : 'en'`
- Exposes `currentLanguage: Signal<'en' | 'de' | null>` for the settings UI
- `setLanguage(lang: 'en' | 'de' | null)` resolves `null` to the OS-detected language (`navigator.language.startsWith('de') ? 'de' : 'en'`), calls `TranslateService.use(resolvedLang)`, and delegates persistence of the original `lang` value (including `null`) to `SettingsService`

### App Initialization

`I18nService` is bootstrapped via `APP_INITIALIZER` in `app.config.ts` — the language is applied before any component renders, preventing a flash of untranslated text.

## Translation File Structure

All keys are namespaced by feature. Interpolation uses `@ngx-translate`'s `{{ param }}` syntax.

```typescript
export interface Translation {
  SETTINGS: {
    TITLE: string;
    SHORTCUT_LABEL: string;
    SHORTCUT_PLACEHOLDER: string;
    SHORTCUT_HINT: string;
    MAX_ENTRIES_LABEL: string;
    MAX_ENTRIES_RANGE: string;       // "entries ({{min}} – {{max}})"
    LANGUAGE_LABEL: string;
    LANGUAGE_SYSTEM: string;
    LANGUAGE_EN: string;
    LANGUAGE_DE: string;
    SAVE: string;
    SAVING: string;
    SAVED: string;
  };
  CLIPBOARD: {
    TITLE: string;
    TAB_RECENT: string;
    TAB_PINNED: string;
    FILTER_ALL: string;
    FILTER_TEXT: string;
    FILTER_IMAGE: string;
    SEARCH_PLACEHOLDER: string;
    ERROR_LOAD: string;
    TRY_AGAIN: string;
    EMPTY_PINNED: string;
    EMPTY_PINNED_HINT: string;
    EMPTY_NO_MATCHES: string;        // "No matches for '{{term}}'"
    EMPTY_NOTHING: string;
    HINT_NAV: string;
    HINT_PASTE: string;
    HINT_DELETE: string;
    HINT_PIN: string;
    HINT_SEARCH: string;
    HINT_CLOSE: string;
  };
  ENTRY: {
    IMAGE: string;
    TOGGLE_PIN: string;
    DELETE: string;
    TIME_JUST_NOW: string;
    TIME_MINUTES: string;            // "{{n}}m ago"
    TIME_HOURS: string;              // "{{n}}h ago"
    TIME_DAYS: string;               // "{{n}}d ago"
  };
  IMAGE_PREVIEW: {
    TITLE: string;
    COPY: string;
    COPYING: string;
    LOADING: string;
    ERROR: string;
    CLOSE: string;
    COPIED: string;
  };
}
```

Language name keys are translated in the current language (e.g. "German" in English, "Deutsch" in German):

| Key | en | de |
|-----|----|----|
| `SETTINGS.LANGUAGE_SYSTEM` | System | System |
| `SETTINGS.LANGUAGE_EN` | English | Englisch |
| `SETTINGS.LANGUAGE_DE` | German | Deutsch |

## Settings UI

The settings component gains a **Language** row above the Save button. Implementation notes:

- Use **Spartan NG `hlm-select`** component — check [spartan.ng/components/select](https://spartan.ng/components/select) for the correct component and API before implementing
- Three options: System (OS default), English, Deutsch — each label uses `| translate`
- Driven by a `linkedSignal` tied to `I18nService.currentLanguage`, consistent with `shortcut` and `maxEntries`
- Changing the dropdown **immediately** switches the displayed language (live preview)
- The new value is queued into the pending save state and persisted when the user clicks **Save**

---

## Language Switching Flow

```
User selects language
  → linkedSignal updates
  → I18nService.setLanguage(lang) called
  → TranslateService.use(lang) fires
  → All translate pipes re-evaluate
  → UI updates instantly (no reload)
  → On Save: SettingsService.saveSettings() persists to DB
```

## Backend Changes

### TypeScript model (`settings.model.ts`)

```typescript
export type Language = 'en' | 'de';

export interface AppSettings {
  shortcut: string;
  maxEntries: number;
  language: Language | null;  // null = not set, use OS default
}
```

### Rust model (`models.rs`)

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum Language {
    En,
    De,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct AppSettings {
    pub shortcut: String,
    pub max_entries: i64,
    pub language: Option<Language>,
}
```

### SQLite (`sqlite_store.rs`)

- **No migration adds a default row** — absence of the `language` key in the settings table means "use OS default"
- `get_settings()`: queries `SELECT value FROM settings WHERE key = 'language'`; returns `None` if row is absent or value fails to deserialize
- `save_settings()`: if `language` is `Some(lang)`, upserts the row; if `None`, deletes the row (`DELETE FROM settings WHERE key = 'language'`)

## Out of Scope

- More than two languages (adding a third requires only a new `.ts` file + enum variant)
- Pluralization rules beyond what `{{n}}` interpolation covers
- Number/date locale formatting beyond relative timestamps
