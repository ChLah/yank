# spartan/ng Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install spartan/ng locally, generate button/input/label/badge/alert components, and replace all hand-rolled Tailwind controls across four Angular components with their spartan equivalents.

**Architecture:** spartan/ng uses a two-layer model — `@spartan-ng/brain` provides headless Angular CDK primitives, the CLI generates styled "helm" wrappers locally under `src/libs/ui/`. Directives (`hlmBtn`, `hlmInput`, `hlmBadge`) are applied to existing HTML elements; components (`HlmLabelComponent`, `HlmAlertComponent`) replace their HTML counterparts. Layout classes from the existing Tailwind markup are preserved; only the styling classes that spartan now owns are removed.

**Tech Stack:** Angular v21, Tailwind v4 (PostCSS), spartan/ng (brain + helm), @angular/cdk, pnpm

## Task 2: Run spartan init and generate components

**Files:**
- Modify: `src/styles.css`, `tsconfig.json` (both updated automatically by CLI)
- Create: `src/libs/ui/ui-{button,input,label,badge,alert}-helm/` trees

- [ ] **Step 1: Run spartan init**

```bash
pnpm exec ng g @spartan-ng/cli:init
```

The CLI will prompt. Answer as follows:
- **Where would you like to store your generated components?** → accept the default (`src/libs/ui`)
- Any other prompts → accept defaults

- [ ] **Step 2: Verify styles.css was updated**

Open `src/styles.css`. It must now start with explicit Tailwind layer imports. If the file still begins with `@import "tailwindcss"`, replace the first line manually:

Replace:
```css
@import "tailwindcss";
```

With:
```css
@layer theme, base, components, utilities;
@import "tailwindcss/theme.css" layer(theme);
@import "tailwindcss/preflight.css" layer(base);
@import "tailwindcss/utilities.css";
@import "@spartan-ng/brain/hlm-tailwind-preset.css";
```

Keep all existing `@layer base` and `@layer utilities` blocks below unchanged. Final `src/styles.css`:

```css
@layer theme, base, components, utilities;
@import "tailwindcss/theme.css" layer(theme);
@import "tailwindcss/preflight.css" layer(base);
@import "tailwindcss/utilities.css";
@import "@spartan-ng/brain/hlm-tailwind-preset.css";

@layer base {
  :root {
    color-scheme: dark;
  }

  * {
    box-sizing: border-box;
    font-family: 'DM Sans', system-ui, -apple-system, sans-serif;
  }

  html, body {
    margin: 0;
    padding: 0;
    height: 100%;
    background: transparent;
  }
}

@layer utilities {
  .scrollbar-thin {
    scrollbar-width: thin;
    scrollbar-color: #27272a transparent;
  }
}
```

- [ ] **Step 3: Generate all five spartan components in one go**

```bash
pnpm exec ng g @spartan-ng/cli:ui --names button,input,label,badge,alert
```

If `--names` is not supported by your CLI version, run interactively and select all five:

```bash
pnpm spartan
```

Use Space to select: `button`, `input`, `label`, `badge`, `alert`. Then Enter to confirm.

- [ ] **Step 4: Verify generated files exist**

```bash
ls src/libs/ui/
```

Expected output should include directories: `ui-button-helm`, `ui-input-helm`, `ui-label-helm`, `ui-badge-helm`, `ui-alert-helm`.

- [ ] **Step 5: Verify tsconfig.json has path aliases**

Open `tsconfig.json`. The `compilerOptions.paths` section must contain entries like:

```json
    "paths": {
      "@spartan-ng/ui-button-helm": ["src/libs/ui/ui-button-helm/src/index.ts"],
      "@spartan-ng/ui-input-helm": ["src/libs/ui/ui-input-helm/src/index.ts"],
      "@spartan-ng/ui-label-helm": ["src/libs/ui/ui-label-helm/src/index.ts"],
      "@spartan-ng/ui-badge-helm": ["src/libs/ui/ui-badge-helm/src/index.ts"],
      "@spartan-ng/ui-alert-helm": ["src/libs/ui/ui-alert-helm/src/index.ts"]
    }
  }
}
```

If any path is missing, add it manually matching the pattern above.
